Article 2: What is Source Control and Why do you need it?

What is Source Control and Why do you need it?

This article by Joel on Software gives a great beginners approach to Source Control. Joel states that Source Control allows distributed work in teams of any size, at different locations, while avoiding conflicts, in source code changes. Joel then describes the basic terms associated with Source Control: Repository: the area where the all the files live. Revision: allowing comparision of changes and who has made those changes. Working Copy: A local copy each developer can work on without changing the source file. Branching: a branch is a copy of the main revision which can be merged into other branched and the main source file. Merging: allows re-introducing changes down in a branch back to the main copy. Joel then states some of the benefits of Source Control:
  • Easier backups of one, central location
  • Easy development of new features
  • Historical overview of changes
  • Access control for revisions
  • Conflict resolution of competing changes
For me it makes perfect sense why Source Control is imperetive. First of all because if you don't have it you are risking losing all your work! Second, it seems like its super important for having multiple people working on the same project.
